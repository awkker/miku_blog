# Agent Guide Phase 1: Frontend Pages & Detailed Layout (miku-blog-ai)

## 🎯 阶段目标与全局视觉规范
本阶段构建博客的核心展示层，必须实现高度还原的 macOS 拟物化风格与极佳的中文排版阅读体验。

### 1. 全局设计 Token (必须在 CSS / Tailwind 中严格定义)
- **主题色**：定义为 `--miku-color: #39c5bb`。所有强调色、Active 状态、悬浮高亮均需使用此颜色（可配合透明度调整，如 `rgba(102,205,170,0.8)`）。
- **中文字体**：全局首选「霞鹜文楷」(LXGW WenKai / LXGW WenKai Screen)，无前置字体时 Fallback 到系统默认无衬线体。
- **英文字体/代码字体**：代码块与英文标识优先使用 `JetBrains Mono` 或 `Fira Code` 等连字等宽字体。
- **液态玻璃风格 (Glassmorphism)**：
  - 使用：frontend/src/components/LiquidGlassCard.vue和frontend/src/utils/liquidGlass.ts

## 🧩 核心基础组件开发规范

### 1. `LiquidGlassCard.vue` (通用液态玻璃卡片)
- **使用位置**：frontend/src/components/LiquidGlassCard.vue

### 2. `MacTerminalCodeBlock.vue` (macOS 终端风格代码块)
- **视觉结构**：
  - **顶部控制栏 (Header)**：高度约 `32px`，背景色为稍浅的深灰（如 `#2d2d2d`），左侧包含三个极其经典的圆点（关闭红 `#ff5f56`、最小化黄 `#ffbd2e`、全屏绿 `#27c93f`），圆点间距 `8px`。
  - **中间标题**：在控制栏居中显示当前代码的语言名称（如 `TypeScript`），字体微小、灰色。
  - **右上角操作**：放置一个极其低调的“复制代码”按钮（Hover 时显示背景）。
  - **代码内容区 (Body)**：背景色为深邃黑（如 `#1e1e1e`），内部代码必须支持基于 Shiki 或 PrismJS 的语法高亮，字体使用等宽字体，开启横向滚动条隐藏 (`scrollbar-hide`)。

## 🖥️ 页面级实现极致细节规格

### 1. 开屏引导页 (Landing Page) -> `src/pages/index.astro`
- **背景**：固定全屏图，使用 `bg-cover bg-center bg-no-repeat`，禁止产生滚动条。
- **macOS 顶部菜单栏 (Top Bar)**：
  - 高度限制在 `28px - 32px`。
  - `backdrop-blur-md bg-white/20 text-white/90 text-sm font-medium px-4`。
  - 左侧：渲染 Apple 风格 Logo 或纯文本 "Snowykami OS"。
  - 右侧 Flex 布局：包含播放状态("暂无播放")、控制按钮(上一首/暂停/下一首/循环的 SVG 图标)、音量滑块、天气图标、系统日期时间。
- **中央文案区 (Hero Text)**：
  - **主标题**：绝对居中，字号极大约 `6rem`，字体加粗，必须带有多重文字阴影以确保在复杂背景下的可读性 (`drop-shadow-xl`)。
  - **副标题 (打字机效果)**：位于主标题下方，字号 `1.5rem`。文案严格为 `「月が綺麗ですね, 風も優しいですね」。`。
  - **动画要求**：使用 CSS `@keyframes` 或 Vue 的 `onMounted` 结合 `setInterval` 逐字输出，结尾必须有一个闪烁的 `|` 光标 (`animation: blink 1s step-end infinite`)。
- **底部 Dock 栏**：
  - 绝对定位在底部，居中，距离底部 `16px`。
  - 容器外观：极度圆角 `rounded-2xl`，液态玻璃背景 `bg-white/20 backdrop-blur-lg border border-white/30`，四周留有 `padding`。
  - **Dock Item 动效 (仿 macOS)**：图标在 hover 时必须有平滑的放大效果 (`scale-125` 或 `scale-150`)，且相邻图标要有联动的微放大（如果有能力实现的话，否则保证当前 item 放大且有底部小白点提示 active 状态）。图标使用单色细线 SVG。

### 2. 博客主页 (Blog Index) -> `src/pages/blog/index.astro`
- **Hero Banner (顶部视差图)**：
  - 高度约为 `40vh`，背景图自适应。
  - 必须有一层渐变遮罩 `bg-gradient-to-b from-transparent to-white/90`（暗色模式下过渡到背景色），让下方内容平滑过渡。
  - 居中显示博客名（大字）和签名（小字）。
- **文章瀑布流列表 (左侧主干)**：
  - 顶部 Tab 栏：使用幽灵按钮样式，当前选中项的文字或下划线颜色为 `--miku-color`。
  - **文章卡片细节**：
    - 上半部分封面图：高度固定（如 `200px`），`object-cover`，悬浮卡片时，图片自身触发 `scale-105` 缓慢放大。
    - 下半部分信息区：包含分类 Tag（圆角小标签，背景色非常浅的主题色，文字为主题色）、粗体标题（Hover 时变主题色）、两行截断的灰色摘要 `line-clamp-2`、底部日期。
- **侧边栏 (右侧)**：
  - **博主卡片**：顶部放一个圆形头像，设置 `hover:rotate-[360deg] transition-transform duration-700` 转动效果。下方紧接加粗名称、签名。底部是三等分的统计数据（文章数/分类/访问量）。
  - **热榜卡片**：列表渲染，前三名的序号使用高亮颜色（如红、橙、黄），后续序号为灰色。标题单行截断，末尾带上热度数值。

### 3. 博客阅读页 (Article Detail) -> `src/pages/blog/[slug].astro`
- **Markdown 正文排版 (Tailwind Typography/Prose 极致调优)**：
  - 行高 `leading-loose` (约 1.8)，段落间距 `mb-6`。
  - 所有 `h1, h2, h3` 必须带有左侧边框或特殊标识（颜色为 `--miku-color`），且带底部浅色分割线。
  - 引用块 (`<blockquote>`)：左侧粗边框为主题色，背景为极浅的主题色 (`bg-[rgba(102,205,170,0.05)]`)，文字斜体。
  - **代码块**：全部接管为前面定义的 `MacTerminalCodeBlock.vue`。
- **左侧悬浮互动栏 (Sticky Left Sidebar)**：
  - 距离屏幕顶部 `100px` 悬浮。
  - 卡片宽度较窄，内部元素纵向排列。
  - 点赞按钮：点击时必须有“心跳”跳动动画 (`scale-125` 瞬间回弹)，且数字+1，图标变为实心主题色。
- **右侧目录导航栏 (Sticky Right TOC)**：
  - 根据滚动条位置，动态高亮当前阅读章节。
  - 高亮状态：文字变为主色 `--miku-color`，且左侧有一个绝对定位的主色指示小竖线平滑移动。未选中状态为低饱和度灰色。

## 🚦 AI 编码执行指令
1. 建立 `tailwind.config.ts` 时，必须先把上述所有的颜色、字体、阴影、动画 Keyframes 配置完整。
2. 开发组件时，严禁使用内联样式 (`style="..."`)，必须全部使用 Tailwind 类名。
3. 遇到动画和交互状态，优先使用 CSS 过渡 (`transition` / `group-hover`)，Vue 的 `ref` 仅用于复杂逻辑（如监听滚动条控制 TOC 高亮）。
4. 在此阶段，必须使用高质量的网图 URL 和精致的假文案把页面结构撑起来，禁止留白。
