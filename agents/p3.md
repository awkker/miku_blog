# Local Agent Guide (miku-blog-admin)

本指南是 `miku-blog` 后台管理系统（Admin Dashboard）的专属开发约束。在生成或修改后台代码时，必须同时遵守全局 `agents.md` 与本局部指南。

## 📐 布局与架构约束 (Layout & Architecture)

### 经典后台布局
- **整体结构**：必须采用“左侧固定侧边栏 (Sidebar) + 顶部状态栏 (Header) + 右侧主内容区 (Main Content)”的经典布局。
- **左侧边栏 (Sidebar)**：
  - 必须固定在屏幕左侧，包含站点的完整导航菜单。
  - 激活项 (Active Item) 必须使用全局主色 `--miku-color` (#39c5bb) 进行视觉突出。
- **顶部状态栏 (Header)**：
  - 必须包含管理员头像 (Avatar) 和当前页面标题/面包屑。
  - 右侧可预留快捷操作（如：退出登录、暗黑模式切换、通知中心）。

### 技术实现细节
- **状态驱动**：后台属于重度交互区域，页面主体结构（Sidebar, Header）和数据图表必须作为 Vue 3 互动岛屿 (Islands) 挂载。
- **Tailwind 优先**：严格使用 Tailwind CSS 原子类进行网格 (Grid)、弹性盒子 (Flex) 的布局排版。

## 🎨 视觉与交互约束 (Admin UI/UX)

### 克制的液态玻璃风格 (Restrained Glassmorphism)
虽然全局要求使用液态玻璃风格，但在后台数据密集型场景中必须**保持克制**，以保证数据的可读性：
- **背景处理**：废弃过于花哨的水彩/插画背景。后台底层背景必须是干净的浅色（如 `bg-slate-50`）或暗色。
- **卡片设计**：
  - 数据面板、表格容器等卡片组件，依然可以使用半透明与模糊效果（`backdrop-blur-md` + `bg-white/40` 等）。
  - **强约束**：卡片内的文字必须保持极高的对比度。禁止在正文阅读区或复杂表格区使用强烈的背景干扰。
- **色彩搭配**：
  - 使用 `#39c5bb` 作为主要数据指标、进度条、主按钮的颜色。
  - 使用薰衣草色（Lavender）作为图表对比数据列、Hover 状态光晕或次要标签的颜色，与主色形成优雅互补。

### 数据可视化 (Data Visualization)
- **图表接入**：使用 ECharts 展示浏览量、评论趋势等数据。
- **图表主题**：图表的线条、柱状图颜色必须从 `#39c5bb` 和薰衣草色中取色，保持全站视觉统一。

## ⚠️ 行为准则补充
- **数据真实感**：在生成后台组件的 mock 数据时，请使用贴合真实博客运营的数据（如：文章阅读量 4,298，待审评论等），不要使用 Lorem Ipsum 占位符。
- **响应式兜底**：后台页面主要面向桌面端 (Desktop-first)，但在设计 Grid/Flex 布局时，必须提供基本的移动端适配（如侧边栏在小屏幕下收起为汉堡菜单）。
