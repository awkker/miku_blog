# 液态玻璃效果实战：从视觉到性能的平衡

液态玻璃效果的难点不在“做出来”，而在“持续可用”。  
如果动画和滤镜没有边界，页面在低端设备上会明显掉帧。

## 先拆层：结构层、视觉层、交互层

我把组件拆成三层：

- 结构层：负责布局和内容插槽
- 视觉层：负责滤镜、折射和高光
- 交互层：负责 hover、聚焦和轻微位移反馈

这样做的好处是，视觉失效时可以单独降级，不影响内容可读性。

## 参数集中配置，避免“魔法数字”

参数最好集中放在一个对象里，便于统一调优：

```ts
const glassOptions = {
  blur: 0.3,
  contrast: 1.14,
  brightness: 1.04,
  saturate: 1.08,
  displacementStrength: 1.08,
}
```

当你需要做 A/B 视觉比较时，这种组织方式会非常省时间。

## 降级策略要明确

真实项目里，`backdrop-filter` 并不总是可用。  
建议准备两个层级：

1. 完整液态效果：支持 SVG displacement + backdrop filter。
2. 轻量玻璃效果：只保留半透明渐变、阴影和边框。

只要保证文字对比度，用户不会因为降级而“无法阅读”。

## 动效节奏：慢一点更高级

玻璃组件的微动效不需要快。  
200ms 的锐利反馈更像按钮，400~700ms 的缓动更像材质变化。

结论很简单：能看懂内容、不卡顿、风格统一，才算“可交付”的玻璃效果。
